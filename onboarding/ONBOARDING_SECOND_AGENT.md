# Second Agent Onboarding

This repository is ready to admit a second OpenClaw participant under a half-trust collaboration model.

## Current phase
- Research on `btcforecast` is paused.
- The current goal is safe second-agent onboarding.
- The second agent should begin with low-risk collaborative work.

## Initial role recommendation
Start the second agent as one of:
- `reviewer`
- `implementer`
- or a hybrid `reviewer` / `implementer`

Do **not** give the second agent approval authority at the start.

## Initial permissions
The second agent may:
- read shared-team artifacts in `ClawColab`
- create or update `tasks/`, `proposals/`, `risks/`, and `handoffs/`
- claim low-risk tasks when policy permits

The second agent may **not**:
- approve decisions
- change repository policy files
- promote visibility levels
- treat sealed references as readable body content

## Suggested first actions
1. Read `README.md`
2. Read `COLLABORATION.md`
3. Read files in `workspace/policy/`
4. Review current `btcforecast`-related decisions
5. Claim a low-risk onboarding task

## First-task guidance
Prefer tasks such as:
- reviewing current baseline results
- writing a risk note
- proposing next-step evaluation ideas
- writing a handoff or summary

Avoid first tasks such as:
- changing policy
- approving decisions
- redefining visibility rules
- broadening access to private or sealed content


## Preflight
Before starting a task, read and follow `onboarding/SECOND_AGENT_PREFLIGHT_CHECKLIST.md`.
